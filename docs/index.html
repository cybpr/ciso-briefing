⚡ ACT NOW CVE-2026-0300 — Palo Alto PAN-OS Root RCE Zero-Day (Active Exploitation)
RPS: 9.07
SOURCE: Palo Alto Networks / CISA KEV
PUBLISHED: May 6, 2026
CVSS: 9.3 (Critical)
RPS CALC: S=9.3 × 0.45 + E=9.5 × 0.35 + X=8.5 × 0.20 = 9.07
PATCH ETA: May 13, 2026
MITRE ATT&CK: T1190, T1068, T1059, T1562.001

CVE-2026-0300 is an unauthenticated remote code execution vulnerability in PAN-OS, carrying a CVSS score of 9.3 when the User-ID Authentication Portal is internet-exposed. Exploitation attempts began as early as April 9, 2026, with attackers successfully achieving RCE and injecting shellcode approximately one week later; following compromise, attackers immediately conducted log cleanup to evade detection. The attackers then deployed tools with root privileges and conducted Active Directory enumeration using the firewall's service account credentials, targeting domain root and DomainDnsZones, before deleting privilege escalation evidence from the audit log. Unit 42 has attributed the observed activity to CL-STA-1132, a threat cluster likely sponsored by a nation-state, whose objective is rarely the firewall itself — after gaining root access, these actors conduct Active Directory enumeration and deploy open-source tunneling tools. Shadowserver is currently tracking more than 5,800 PAN-OS VM-Series firewalls exposed to the public internet, with the largest concentrations in Asia (2,466 instances) and North America (1,998 instances).

Immediate Actions: Restrict User-ID Authentication Portal to trusted internal IPs or disable entirely. Disable Response Pages on all L3 interfaces with internet-facing traffic. Apply Threat ID 510019 detection where PAN-OS ≥ 11.1. Prioritize patch deployment from May 13 hotfix release. Treat any affected firewall as potentially compromised.

ShinyHunters claimed responsibility on May 3, 2026, for a massive data breach targeting Instructure, the parent company of Canvas, alleging access to the personal identifying information of 275 million individuals including students and teachers. The cybercrime group then defaced the login pages of several Instructure customer schools with an extortion message, claiming this was a second, separate breach. Instructure's CISO Steve Proud confirmed the data included "certain identifying information" — including names, email addresses and student ID numbers — as well as messages exchanged by users. Experts warn that with access to real names, email addresses and even teacher-student messages, "the next wave of phishing will not be generic — it will reference real courses and real conversations, which makes it far more likely to succeed." Mandiant earlier this year reported an increase in activity consistent with prior ShinyHunters-branded extortion operations, noting attackers use sophisticated voice phishing and fake company-branded login pages to harvest employee credentials before stealing data from cloud platforms.

Immediate Actions: Inventory Canvas and Instructure integrations across your organization. Alert staff about highly personalized spear-phishing referencing real course/project data. Force password resets for affected users. Confirm MFA is enforced across all connected education platforms. Engage legal on FERPA/GDPR notification obligations if student data applies.
⚡ ACT NOW CVE-2026-31431 — Linux "Copy Fail" Kernel LPE (Cloud/Kubernetes/Container Breakout)
RPS: 8.16
SOURCE: Microsoft Security Blog / CISA KEV
PUBLISHED: May 1, 2026
CVSS: 7.8 (High)
RPS CALC: S=8.5 × 0.45 + E=8.0 × 0.35 + X=9.0 × 0.20 = 8.18
FEDERAL DEADLINE: May 15, 2026
MITRE ATT&CK: T1068, T1611, T1210, T1552

Microsoft Defender is investigating CVE-2026-31431 ("Copy Fail"), a high-severity local privilege escalation vulnerability affecting Red Hat, SUSE, Ubuntu, and AWS Linux that enables root privilege escalation across cloud environments and Kubernetes workloads — with a working exploit already in the wild. The vulnerability resides in the algif_aead module of the Linux kernel's AF_ALG cryptographic subsystem, and alarmingly, a 732-byte Python script is all an unprivileged local user needs to reliably escalate privileges to root. Exploitation requires no root privileges inside containers, no kernel modules, and no network access, making it a powerful post-exploitation tool in containerized environments including Kubernetes clusters and Docker CI runners. Successful exploitation leads to full root privilege escalation and could facilitate container breakout, multi-tenant compromise, and lateral movement within shared environments; its reliability, stealth (in-memory-only modification), and cross-platform applicability make it particularly dangerous in cloud, CI/CD, and Kubernetes environments. CISA added CVE-2026-31431 to its KEV catalog on May 1, 2026, with a mandatory remediation deadline of May 15, 2026; patches are available in Linux kernel versions 6.18.22, 6.19.12, and 7.0.

Immediate Actions: Audit Linux kernel versions across all cloud workloads, container environments, and on-premises hosts. Apply kernel patches immediately. Where patching is delayed, block AF_ALG socket creation and limit setuid binaries. Treat any container RCE finding as a potential full host compromise; enforce rapid node recycling.

Ivanti released fixes for five high-severity vulnerabilities in its Endpoint Manager Mobile (EPMM) solution on May 8, 2026, one of which — CVE-2026-6973 — has been exploited as a zero-day; CVE-2026-6973 is caused by improper input validation and allows remote attackers with administrative privileges to execute arbitrary code on vulnerable instances. Ivanti noted that customers who followed its January recommendation to rotate credentials after exploitation of CVE-2026-1281 and CVE-2026-1340 have significantly reduced risk from this new flaw — both of those earlier CVEs were also exploited as zero-days before Ivanti was aware of them. There are currently no reliable atomic indicators of compromise for CVE-2026-6973; CISA has added it to the Known Exploited Vulnerabilities catalog and ordered U.S. federal civilian agencies to remediate within three days. EPMM is a high-profile target given the privileged position attackers gain on the device, which may allow for lateral movement within the compromised network, and the product has been repeatedly targeted by zero-day vulnerabilities.

Immediate Actions: Update EPMM to versions 12.6.1.1, 12.7.0.1, or 12.8.0.1 immediately. If previously exploited via CVE-2026-1281/1340, rotate ALL admin credentials now. Review EPMM-adjacent Sentry appliance security configuration. Assume no reliable IoCs — hunt for anomalous admin activity as your primary detection method.
