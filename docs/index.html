The exploited SharePoint Server vulnerability (CVE-2026-32201) has been described as a spoofing issue with a CVSS score of 6.5; improper input validation allows an unauthorized attacker to perform spoofing over a network, potentially enabling access to sensitive information and the ability to alter it. Iranian IRGC-CEC / CyberAv3ngers Actively Disrupting U.S. Critical Infrastructure PLCs (CISA AA26-097A)

Source: CISA Advisory AA26-097A  |  Published: April 7, 2026  |  MITRE ATT&CK for ICS: T0883 (Internet Accessible Device), T0885 (Commonly Used Port), T0828 (Loss of Productivity and Revenue), T0816 (Device Restart/Shutdown), T0836 (Modify Parameter)

On April 7, 2026, the FBI, CISA, NSA, EPA, DOE, and U.S. Cyber Command's Cyber National Mission Force jointly released advisory AA26-097A, warning of active exploitation of internet-exposed Rockwell Automation PLCs by Iranian-affiliated cyber actors linked to the IRGC Cyber Electronic Command, operating as CyberAv3ngers — also tracked as Shahid Kaveh Group, Storm-0784, and UNC5691. Since at least March 2026, these actors have disrupted PLCs deployed across multiple U.S. critical infrastructure sectors, including Government Services, Water and Wastewater Systems, and the Energy sector, with some victims experiencing operational disruption and financial loss. The attacks caused operational disruption and financial loss through tampering with vulnerable HMIs and SCADA systems, using Rockwell's own Studio 5000 Logix Designer software to achieve their goals. The group has escalated from exploiting default credentials on Israeli-made PLCs in 2023, to deploying the IOCONTROL ICS malware platform in 2024, to actively exploiting CVE-2021-22681, a critical authentication bypass in Rockwell Automation controllers, in 2026 — and there is no vendor patch for this vulnerability; only defense-in-depth mitigations are available. Censys identifies 5,219 internet-exposed hosts globally responding to EtherNet/IP, with 74.6% concentrated in the United States.

Apache ActiveMQ CVE-2026-34197 — 13-Year Latent RCE Flaw Actively Exploited, 6,300+ Exposed Instances (CISA KEV)

Source: BleepingComputer & GBHackers  |  Published: April 17–21, 2026  |  MITRE ATT&CK: T1190 (Exploit Public-Facing Application), T1059.007 (JavaScript), T1071 (Application Layer Protocol), T1210 (Exploitation of Remote Services)

CVE-2026-34197, a security flaw that went undetected for 13 years, was discovered by a Horizon3 researcher using the Claude AI assistant; it stems from improper input validation, allowing authenticated threat actors to execute arbitrary code via injection attacks. CVE-2026-34197 can be chained with an older vulnerability (CVE-2024-32114) to achieve fully unauthenticated remote code execution. Threat actors are actively targeting exposed Jolokia management endpoints in Apache ActiveMQ Classic deployments, with Fortinet FortiGuard Labs telemetry uncovering dozens of exploitation attempts, peaking on April 14, 2026. Over 6,000 internet-facing Apache ActiveMQ servers are currently affected by this critical security flaw, with 6,364 vulnerable IPs identified by the Shadowserver Foundation on April 19, 2026. CISA added CVE-2026-34197 to its Known Exploited Vulnerabilities catalog and ordered Federal Civilian Executive Branch agencies to patch ActiveMQ servers by April 30. Initial attacks appear focused on establishing initial access to corporate networks, with subsequent deployment of ransomware, cryptocurrency miners, and backdoors.

Fortinet released out-of-band patches for a critical security flaw impacting FortiClient EMS (CVE-2026-35616, CVSS 9.1) — described as a pre-authentication API access bypass leading to privilege escalation — which allows an unauthenticated attacker to execute unauthorized code or commands via crafted requests.
