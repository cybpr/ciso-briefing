🔴 ACT NOW  |  #1 — Linux Kernel "Copy Fail" (CVE-2026-31431): Universal LPE, CISA KEV, Active Exploitation RPS: 9.2

CVE / CVSS: CVE-2026-31431  |  CVSS 7.8 (High)
MITRE ATT&CK: T1068 – Exploitation for Privilege Escalation  |  T1611 – Escape to Host

Security researchers at Theori disclosed CVE-2026-31431, dubbed "Copy Fail," a high-severity local privilege escalation (LPE) vulnerability in the Linux kernel affecting virtually every major Linux distribution shipped since 2017, with a working proof-of-concept exploit publicly available. Successful exploitation leads to full root privilege escalation and could facilitate container breakout, multi-tenant compromise, and lateral movement within shared environments — its reliability, stealth via in-memory-only modification, and cross-platform applicability make it particularly dangerous in cloud, CI/CD, and Kubernetes environments. CISA added CVE-2026-31431 to its KEV catalog on May 1, 2026, following confirmed attacks against U.S. think tanks, European manufacturing companies, and cloud service providers — attackers typically pair Copy Fail with an RCE bug to obtain initial local access, then escalate to deploy ransomware or exfiltrate data. Unlike Dirty Cow and Dirty Pipe, Copy Fail can be exploited without winning a race condition, and the same exploit will work reliably across many systems. Immediate mitigation: apply kernel patches or blacklist the algif_aead module; CISA mandates federal patching by May 15, 2026.

🔴 ACT NOW  |  #2 — Mini Shai-Hulud Supply Chain Worm: PyTorch Lightning PyPI Compromise & Credential Theft RPS: 8.9
Source: The Hacker News  |  Socket.dev  |  Semgrep
Published: April 30, 2026
RPS Calculation: (S:9 × 0.45) + (E:9 × 0.35) + (X:9 × 0.20) = 4.05 + 3.15 + 1.80 = 8.90
CVE: No CVE assigned at time of publication
MITRE ATT&CK: T1195.001 – Compromise Software Supply Chain  |  T1552.001 – Credentials in Files  |  T1059 – Command and Scripting Interpreter

Threat actors published two malicious versions (2.6.2 and 2.6.3) of the popular PyTorch Lightning PyPI package on April 30, 2026, assessed to be an extension of the Mini Shai-Hulud supply chain incident that had previously targeted SAP-related npm packages. The malicious versions contain a hidden _runtime directory with an obfuscated JavaScript payload that executes automatically upon module import, stealing credentials, authentication tokens, environment variables, and cloud secrets, while also attempting to poison GitHub repositories. The Lightning package receives hundreds of thousands of downloads per day and millions per month, making this among the highest-impact PyPI compromises of the year. The broader TeamPCP supply chain campaign has also compromised Checkmarx's KICS Docker images and Bitwarden's CLI npm package; between April 21–23, three separate supply chain attacks hit npm, PyPI, and Docker Hub, all targeting API keys, cloud credentials, SSH keys, and tokens from developer environments and CI/CD pipelines. Immediately audit any environment where Lightning 2.6.2/2.6.3 was installed; rotate all potentially exposed GitHub, npm, AWS, Azure, and GCP credentials; pin to Lightning 2.6.1.

MITRE ATT&CK: T1190 – Exploit Public-Facing Application  |  T1078 – Valid Accounts  |  T1548 – Abuse Elevation Control Mechanism

A critical authentication bypass vulnerability (CVE-2026-41940) in cPanel was confirmed being exploited in the wild, with attackers spotted exploiting it since at least February 23 — likely even earlier — well before public disclosure. CVE-2026-41940 stems from missing authentication for a critical function, allowing unauthenticated remote attackers to gain unauthorized access to the control panel. Successful exploitation grants an attacker full control over the cPanel host system, its configurations and databases, and all websites it manages — Shodan shows approximately 1.5 million cPanel instances exposed to the internet. The Shadowserver Foundation reports 44,000 unique IPs actively scanning and running exploits, with approximately 650,000 exposed cPanel/WHM instances observed. Patch to the fixed version immediately; the vendor published the security advisory on April 28, 2026, with updates released shortly after.

MITRE ATT&CK: T1190 – Exploit Public-Facing App  |  T1505.003 – Web Shell  |  T1574.002 – DLL Side-Loading  |  T1003 – Credential Dumping (Mimikatz)

Cybersecurity researchers disclosed details of a China-aligned espionage campaign targeting government and defense sectors across South, East, and Southeast Asia, along with one European government belonging to NATO. The threat cluster, tracked as SHADOW-EARTH-053, exploits vulnerabilities in Microsoft Exchange Server and IIS systems to deploy ShadowPad malware and web shells, with targeted countries including India, Thailand, Malaysia, Myanmar, Sri Lanka, Taiwan, Pakistan, and Poland. Victims also include defense-adjacent IT consulting providers and transportation organizations, indicating a mix of direct and supply-chain targeting; nearly half were previously compromised by a related intrusion set, SHADOW-EARTH-054, which shares identical tool hashes and overlapping techniques. To facilitate privilege escalation, SHADOW-EARTH-053 uses Mimikatz, while lateral movement is accomplished using a custom RDP launcher and a C# implementation of SMBExec known as Sharp-SMBExec. Organizations with internet-facing Exchange or IIS infrastructure should immediately audit patch levels and deploy IPS/WAF virtual patches.
