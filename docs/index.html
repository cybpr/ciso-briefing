Microsoft addressed 164 vulnerabilities in its April 2026 security update release — double the March volume — including one exploited zero-day vulnerability, one previously disclosed zero-day vulnerability, and eight Critical vulnerabilities. This release fixes 169 vulnerabilities including the exploited SharePoint CVE-2026-32201, and represents the second-biggest Patch Tuesday ever, just below the record set in October 2025.

CVE-2026-32201 is an actively exploited zero-day spoofing vulnerability in SharePoint Server (CVSS 6.5) that allows unauthenticated remote attackers to perform spoofing via improper input validation — with no user interaction required. A successful exploit could allow an attacker to view sensitive information and make changes to disclosed information, impacting both confidentiality and integrity. CVE-2026-32201 has been added to CISA's KEV catalog, with federal agencies instructed to patch by April 28.

CVE-2026-33824, a Critical RCE in Windows IKE Service Extensions (CVSS 9.8), allows unauthenticated remote attackers to execute arbitrary code by exploiting a double-free flaw — with low attack complexity and no user interaction required. An unauthenticated attacker exploits it by sending specially crafted packets to a Windows machine with IKE version 2 enabled. Organizations unable to patch immediately can block inbound traffic on UDP ports 500 and 4500, or configure firewall rules to allow IKE traffic only from known peer addresses.

Huntress warns that threat actors are exploiting three vulnerabilities in Microsoft Defender codenamed BlueHammer, RedSun, and UnDefend — all released as zero-days by researcher "Chaotic Eclipse." While BlueHammer and RedSun are local privilege escalation flaws, UnDefend can be used to trigger a denial-of-service condition and block definition updates. Security leadership should also prioritize CVE-2026-33826 (Active Directory RCE, CVSS 8.0), which "represents a direct threat to identity infrastructure" — where an authenticated attacker can execute arbitrary code within the domain and the low complexity of exploitation significantly amplifies risk for potential full domain takeover.

On April 7, 2026, the FBI, CISA, NSA, EPA, DOE, and U.S. Cyber Command's CNMF jointly issued advisory AA26-097A, warning of active exploitation of internet-exposed Rockwell Automation PLCs by Iranian-affiliated cyber actors — specifically CyberAv3ngers, also tracked as Shahid Kaveh Group, Hydro Kitten, Storm-0784, and UNC5691 — targeting PLCs deployed in U.S. government facilities, water and wastewater systems, and energy sector infrastructure.

The advisory explicitly links this escalation to U.S.-Israel military strikes against Iran beginning February 28, 2026. Iran's cyber response was immediate: the Handala group — another IRGC-linked operation — struck medical technology company Stryker with a wiper attack. Actors used leased, third-party hosted infrastructure with Rockwell's Studio 5000 Logix Designer software to create accepted connections to victim PLCs, targeting CompactLogix and Micro850 devices. They deployed Dropbear SSH on victim endpoints for remote access and manipulated HMI/SCADA displays.

The group has escalated from exploiting default credentials on Israeli-made PLCs in 2023 to deploying IOCONTROL malware in 2024, to actively exploiting CVE-2021-22681 — a critical authentication bypass in Rockwell Automation controllers — across U.S. water, energy, and government facilities in 2026. There is no vendor patch for this vulnerability; only defense-in-depth mitigations are available.

Censys identified 5,219 internet-exposed hosts globally responding to EtherNet/IP (port 44818) and self-identifying as Rockwell Automation/Allen-Bradley devices, with the United States accounting for 74.6% of global exposure. Priority defensive steps include removing or tightly restricting internet access to OT/ICS systems, enforcing phishing-resistant MFA, and implementing granular microsegmentation to prevent lateral movement to critical assets.
