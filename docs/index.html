2. Linux Kernel "Copy Fail" LPE — CVE-2026-31431 (CVSS 7.8)

8.46
RPS = (7.8×0.45)+(9×0.35)+(8×0.20)
📰 Xint.io Research  |  📰 Microsoft Security Blog  |  📅 April 29 – May 1, 2026  |  🏷️ MITRE ATT&CK: T1068 (Exploitation for Privilege Escalation), T1611 (Escape to Host), T1543 (Create or Modify System Process)

Copy Fail (CVE-2026-31431) is a logic bug in the Linux kernel's authencesn cryptographic template that lets an unprivileged local user trigger a deterministic 4-byte write into the page cache of any readable file; a 732-byte Python script can obtain root on essentially all Linux distributions shipped since 2017. Unlike previous LPE flaws such as Dirty Cow, Copy Fail is a straight-line logic flaw that triggers without races, retries, or crash-prone timing windows. Successful exploitation leads to full root privilege escalation and could facilitate container breakout, multi-tenant compromise, and lateral movement; its reliability, stealth, and cross-platform applicability make it particularly dangerous in cloud, CI/CD, and Kubernetes environments. While exploitation requires local code execution as a regular user, chaining it with any web RCE, SSH foothold, or malicious CI runner pull request yields immediate root. Microsoft Defender observed preliminary testing activity and, given the availability of a fully working proof-of-concept, expects increased threat actor exploitation within days, consistent with the vulnerability's addition to the CISA KEV catalog.

The PyPI package lightning, a widely used deep learning framework, was compromised in a supply chain attack affecting versions 2.6.2 and 2.6.3 published on April 30, 2026, with malicious code designed to execute credential-stealing malware on import. The threat actor compromised the project's PyPI publishing channel: "An attacker with access to our PyPI credentials cloned our open source code, injected a malicious payload, and pushed those tampered builds directly to PyPI as malicious versions 2.6.2 and 2.6.3, bypassing source control entirely." Any user who ran import lightning with the compromised versions may have had AWS/GCP/Azure cloud credentials, .env files, API keys, browser credentials, and GitHub tokens exfiltrated. If npm publish credentials were found, the worm injected a dropper into every package the token could publish to, bumped the patch version, and republished — causing downstream developers who install those packages to run the full malware. This is consistent with Team PCP's escalating open-source supply chain campaign, which previously compromised LiteLLM, Telnyx, and Xinference in rapid succession.
