CVE-2026-33824 — Windows IKE Service Extensions Critical RCE (CVSS 9.8)

RPS Score

8.83

S:10 × 0.45 + E:8 × 0.35 + X:9 × 0.20

Source: Tenable / CrowdStrike / Zero Day Initiative Published: April 14, 2026 CVSS: 9.8 (Critical) Status: Not yet exploited | Exploitation more likely MITRE ATT&CK: T1190 (Exploit Public-Facing App) · T1210 (Exploitation of Remote Services) · T1021 (Remote Services)

CVE-2026-33824 is a critical remote code execution vulnerability affecting Windows Internet Key Exchange (IKE) Service Extensions with a CVSS score of 9.8. It allows unauthenticated remote attackers to execute arbitrary code by exploiting a double free flaw (CWE-415) in the Windows IKE Extension. No user interaction is required and attack complexity is low — an unauthenticated attacker could exploit this vulnerability by sending specially crafted packets to a Windows machine with IKE version 2 enabled, which could enable remote code execution on the target system.

Affected systems include Windows Server 2025; Windows Server 2022, 23H2 edition; Windows Server 2019 and 2016 with server core installation; and certain desktop versions of Windows 10 and 11 for x64, 32-bit, and ARM systems. Windows admins who cannot immediately install the security update can take one of two actions: block inbound traffic on UDP ports 500 and 4500 for systems that do not use IKE; or, for systems that require IKE, configure firewall rules to allow inbound traffic only from known peer addresses.

From a strategic perspective, leadership should prioritize remediation around this IKE issue and CVE-2026-33826 (Active Directory RCE), due to their combination of exploitability and enterprise-wide impact. The Active Directory hole "represents a direct threat to identity infrastructure" — by exploiting improper input validation in RPC handling, an authenticated attacker can execute arbitrary code within the domain, with the low complexity of exploitation combined with the central role of Active Directory significantly amplifying risk.

This flaw lies in insufficient granularity of access control in Microsoft Defender, which allows an authorized attacker to elevate privileges locally. The description of the flaw matches that of BlueHammer, a zero-day exploit code released publicly earlier in April.
