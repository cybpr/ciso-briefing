CVE-2026-31431, dubbed "Copy Fail," is a local privilege escalation flaw in the Linux kernel's authencesn cryptographic template introduced via code changes across 2011–2017 that nobody connected for nearly a decade. An unprivileged local user can write four controlled bytes into the page cache of any readable file, effectively altering binaries in memory without triggering filesystem-level defenses like inotify. A 10-line Python proof-of-concept script achieves root shell on all major distributions since kernel 4.14. Beyond server LPE, Copy Fail functions as a Kubernetes container escape primitive — because the page cache is shared across container boundaries, a compromised container can corrupt setuid binaries on the host kernel. Discovered using AI-assisted analysis, it was patched in mainline April 1 and added to CISA's KEV. Debian, Ubuntu, SUSE, and others have issued patches; Red Hat initially deferred but reversed course.
