Linux Kernel "Copy Fail" — Universal LPE / Container Escape on All Major Distros Since 2017

📰 Xint (Original Research)  |  Microsoft Security Blog  |  Unit 42  |  Published: April 29 – May 7, 2026
MITRE ATT&CK:
T1068 – Exploitation for Privilege Escalation
T1611 – Escape to Host (container)
T1222 – File and Directory Permissions Modification
Affected Platforms:
All Linux kernels 4.14–6.18.21 / 6.19.11
Ubuntu 24.04 LTS, Amazon Linux 2023, RHEL 10.1, SUSE 16
Kubernetes clusters, Docker, CI/CD runners
Patch / Mitigations:
Upgrade to kernel ≥6.18.22 or ≥6.19.12
Interim: blacklist algif_aead module
CISA KEV deadline: May 15, 2026

Copy Fail lets an unprivileged local user trigger a deterministic, controlled 4-byte write into the page cache of any readable file on a Linux system — a single 732-byte Python script can edit a setuid binary and obtain root on essentially all Linux distributions shipped since 2017. Unlike Dirty Cow and Dirty Pipe, Copy Fail can be exploited without having to win a race condition, and the same exploit will work on many systems. The vulnerability requires local code execution as a regular user, but "chain it with anything that gives you that — web RCE landing in an unprivileged service account, an SSH foothold, a malicious PR on a CI runner — and you're root." Although active exploitation has been primarily observed in proof-of-concept testing, the vulnerability's broad applicability has caused widespread concern, and Microsoft Defender is seeing preliminary testing activity likely to result in increased threat actor exploitation. Priority patching of all cloud-hosted Linux workloads, Kubernetes nodes, and CI/CD runners is required before May 15.
