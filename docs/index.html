CISA added a critical flaw in Ivanti Endpoint Manager Mobile (EPMM) — CVE-2026-1340, a code injection vulnerability that allows an attacker to achieve remote code execution without authentication — to its Known Exploited Vulnerabilities catalog on April 8, with a federal remediation deadline of April 11. The two flaws CVE-2026-1281 and CVE-2026-1340 are typically chained together, allowing a remote, unauthenticated attacker to achieve arbitrary code execution on the EPMM appliance. Mobile device management solutions like Ivanti EPMM are particularly high-value targets because these systems hold elevated privileges on corporate smartphones and tablets; a compromised server could allow attackers to alter security policies or push malicious configurations to thousands of employee devices simultaneously. According to researcher Simo Kohonen, CEO of Defused: "It's been repeatedly exploited literally thousands of times since it was disclosed."

An unauthorized party gained access to Nextend's update infrastructure and distributed a fully attacker-authored build through the official update channel; any site that updated to version 3.5.1.35 between its release on April 7, 2026, and its detection approximately 6 hours later received a fully weaponized remote access toolkit. Patchstack described this as "a textbook supply chain compromise, the kind that renders traditional perimeter defenses irrelevant" — because when the malicious code is delivered through the trusted update channel, "the plugin is the malware."

7.72
RPS  |  S:8.5 · E:8.0 · X:6.5
Source The Hacker News / Security Affairs / Adobe PSIRT
Link https://thehackernews.com/2026/04/adobe-patches-actively-exploited.html
Published April 12, 2026 (Exploitation began ~December 2025)
CVSS 8.6 (revised from 9.6)
MITRE ATT&CK T1204.002 Malicious File · T1059.007 JavaScript · T1005 Data from Local System · T1041 Exfiltration Over C2

Adobe acknowledged that CVE-2026-34621 is being exploited in the wild; researcher Haifei Li disclosed zero-day exploitation of the flaw to run malicious JavaScript code when opening specially crafted PDF documents, with evidence suggesting the vulnerability may have been under exploitation since December 2025. The vulnerability involves prototype pollution — a JavaScript security flaw that permits an attacker to manipulate an application's objects and properties — which could result in arbitrary code execution. The exploit uses util.readFileIntoStream() to read local files and calls RSS.addFeed() to send stolen data to a remote server and receive additional malicious JavaScript, letting attackers profile victims, steal data, and decide whether to escalate with RCE or a sandbox escape. The issue affects Acrobat DC and Reader DC versions prior to 26.001.21411 across Windows and macOS; experts warn these vulnerabilities are particularly dangerous because they require minimal user interaction — often just opening an infected document.
